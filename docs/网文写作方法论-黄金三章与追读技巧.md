# 网文写作方法论：黄金三章与追读技巧

> 本文档汇总中国网文行业（起点中文网、番茄小说、晋江文学城等平台）公开的编辑培训内容、大神作者分享、以及业内广泛认可的写作方法论。目的是提炼可嵌入 AI 写作 prompt 的**具体、可执行规则**。

---

## 一、黄金三章（Golden Three Chapters）原则

### 1.1 定义

"黄金三章"是网文行业最核心的开篇法则。指**小说前三章（约6000-9000字）**必须完成对读者的"锁定"——让读者产生继续阅读的强烈欲望。

**底层逻辑**：网文读者在免费/付费平台上有海量选择，编辑签约审稿时通常只看前三章。如果前三章抓不住人，读者直接划走，编辑直接拒稿。

### 1.2 核心规则（六大铁律）

| # | 规则 | 具体要求 |
|---|------|----------|
| 1 | **第一章必有钩子** | 开篇500字内必须出现冲突、悬念或异常事件。禁止用日常描写、设定介绍、世界观铺垫开头。 |
| 2 | **三章内明确"核心卖点"** | 读者必须在三章内知道这本书"爽在哪里"。是打脸？是升级？是逆袭？是甜宠？核心卖点必须在三章内兑现至少一次。 |
| 3 | **三章内完成首个小高潮** | 必须有一个完整的"冲突-对抗-胜利/翻转"循环。不能三章全在铺垫。 |
| 4 | **主角三章内必须"做事"** | 主角不能是被动的旁观者。第一章就要展示主角的行动力、决策力或独特能力。 |
| 5 | **三章内建立情感锚点** | 读者必须对主角产生认同感（共情/崇拜/好奇）。常用手段：给主角一个让读者心疼的处境 + 展示主角不屈的态度。 |
| 6 | **每章结尾必有悬念** | 前三章的每一章结尾都要留钩子，让读者"忍不住翻下一章"。 |

### 1.3 黄金三章的结构模板

```
第一章（约2000-3000字）：
├── 开篇即冲突：穿越/重生/觉醒/危机/侮辱等"起始事件"
├── 快速建立主角身份 + 核心设定（不超过3句话的交代）
├── 展示主角的第一个决策或行动
└── 章末钩子：更大的危机/秘密/机遇浮现

第二章（约2000-3000字）：
├── 主角开始应对挑战，展示独特能力/金手指
├── 引入第一个对手或阻力
├── 初步展示"爽点"（小胜利、小反杀、小装逼）
└── 章末钩子：新的更大挑战或反转

第三章（约2000-3000字）：
├── 完成第一个小高潮（打脸/逆袭/突破/真相揭露）
├── 给读者"爽到"的感觉（核心卖点的首次完整兑现）
├── 展开更大的故事格局（让读者意识到后面还有更多精彩）
└── 章末钩子：引出第二个大悬念/新目标
```

### 1.4 黄金三章的"禁忌清单"

1. **禁止大段设定介绍**：世界观不要在前三章集中介绍，融入到行动和对话中
2. **禁止主角"旁观"超过半章**：主角必须是推动剧情的主体
3. **禁止无冲突的日常描写**：前三章的每个场景都必须有紧张感或冲突
4. **禁止人物出场超过5个**：前三章只出现核心角色（主角+1-2个关键配角+1个对手）
5. **禁止回忆超过500字**：前三章不做长篇前情回顾
6. **禁止"慢热"开头**：如"某某是一个普通人"这样的平淡叙述

## 二、章章精彩的核心技法

### 2.1 每章必备四要素（CHST框架）

每一章都必须包含以下四个要素中的至少三个：

| 要素 | 英文 | 定义 | 实操标准 |
|------|------|------|----------|
| **冲突** | Conflict | 角色之间、角色与环境之间的对抗 | 每章至少1个明确冲突 |
| **钩子** | Hook | 让读者想继续读的悬念或期待 | 每章结尾必须有钩子 |
| **爽点** | Satisfaction | 让读者获得情绪满足的情节 | 每2-3章至少1个爽点 |
| **转折** | Twist | 出乎意料的发展或信息揭露 | 每3-5章至少1个意外转折 |

### 2.2 章节结构的"三段式"

```
开头（前200-300字）：
├── 承接上一章的钩子
├── 快速进入本章核心场景
└── 1-2句话建立紧张感/期待感

中段（主体1500-2000字）：
├── 核心事件推进
├── 冲突升级或情感深化
├── 信息释放（推进读者理解+制造新疑问）
└── 至少一个"微爽点"或"情绪拐点"

结尾（最后200-300字）：
├── 本章核心事件的阶段性结果
├── 留下未完成的悬念
└── 暗示下一章的爆发或转折
```

### 2.3 章末钩子的12种类型

| # | 钩子类型 | 示例 |
|---|----------|------|
| 1 | **危机悬停** | 主角陷入致命危险，生死未卜 |
| 2 | **新信息炸弹** | 突然揭露一个改变格局的秘密 |
| 3 | **强敌出场** | 更强的对手登场，远超预期 |
| 4 | **反转结尾** | 看似胜利实则陷入更大困局 |
| 5 | **抉择困境** | 主角面临两难选择，读者想知道结果 |
| 6 | **伏笔触发** | 之前埋下的伏笔突然显现端倪 |
| 7 | **时限压力** | "还有三天时间"/"天亮之前必须…" |
| 8 | **情感爆发前奏** | 积蓄的情绪即将爆发但被打断 |
| 9 | **身份暴露** | 主角的秘密身份/能力即将被发现 |
| 10 | **奖励预告** | 巨大的机缘/宝物即将获取 |
| 11 | **误会加深** | 角色之间的误解即将引发更大冲突 |
| 12 | **格局升级** | 突然展现更大的世界/更高的层次 |

## 三、追读（Reader Retention）核心原则

### 3.1 追读的本质

追读 = 读者每天都来看更新。网文的商业模式决定了追读率是最核心的指标。

**追读公式**：追读率 = f(期待感 × 满足感 × 连续性)

### 3.2 追读十大法则

| # | 法则 | 具体说明 |
|---|------|----------|
| 1 | **永远不让读者满足** | 每解决一个问题，立刻抛出新问题。读者"刚爽完就想要下一口" |
| 2 | **每次更新必有进展** | 读者读完必须感觉"今天有东西发生了"，禁止纯水字数 |
| 3 | **短期+长期双线悬念** | 短期悬念（1-3章解决）维持紧迫感，长期悬念（30-100章解决）维持追读动力 |
| 4 | **节奏感 = 张弛有度** | 高潮3-5章 → 短暂过渡1-2章 → 新高潮。不能连续高潮也不能长期平淡 |
| 5 | **每10章一个中高潮** | 大约每10章要有一个"标志性事件"（突破/打脸/获宝/逆转） |
| 6 | **每30章一个大高潮** | 每30章左右要有一个"阶段性大结局感"（副本完结/大敌打败/身份揭露） |
| 7 | **断章艺术** | 在最精彩的地方"今天就更到这里"，读者第二天一定回来 |
| 8 | **角色成长可感知** | 读者要能清晰感受到主角在变强/变聪明/关系在推进 |
| 9 | **承诺与兑现** | 前面暗示的东西必须兑现（但可以用意想不到的方式） |
| 10 | **情绪过山车** | 让读者在一次阅读中经历多种情绪（紧张→放松→愤怒→痛快→感动） |

### 3.3 导致掉追读的致命错误

1. **连续3章以上无爽点**：读者耐心极限
2. **主角连续挨打/受辱不反击**：超过2章不反击，读者弃书
3. **大段无关紧要的对话/描写**：读者感到浪费时间
4. **新角色/设定涌入过多**：读者认知过载
5. **主角智商下线**：做出明显愚蠢的决定
6. **剧情重复**：相似套路连续使用超过2次
7. **断在平淡处**：章末没有悬念，读者没有回来的理由

## 四、读者弃书的常见原因（Top 15）

### 4.1 内容层面

| # | 弃书原因 | 具体表现 | 严重程度 |
|---|----------|----------|----------|
| 1 | **开头无聊** | 前三章没有冲突或悬念 | ⭐⭐⭐⭐⭐ |
| 2 | **主角无魅力** | 性格模糊、行为被动、没有独特之处 | ⭐⭐⭐⭐⭐ |
| 3 | **节奏拖沓** | 一件小事写3-5章，注水严重 | ⭐⭐⭐⭐⭐ |
| 4 | **主角智商下线** | 有明显更好的解决方案却偏选最蠢的 | ⭐⭐⭐⭐ |
| 5 | **人设崩塌** | 角色行为与前文建立的性格矛盾 | ⭐⭐⭐⭐ |
| 6 | **爽点缺失** | 长期压抑不释放，读者得不到满足 | ⭐⭐⭐⭐ |
| 7 | **逻辑漏洞** | 设定自相矛盾、剧情不合理 | ⭐⭐⭐⭐ |
| 8 | **后宫/感情线混乱** | 女主过多、感情线引发读者不适 | ⭐⭐⭐ |
| 9 | **大段设定/说教** | 停下剧情大段介绍世界观或道理 | ⭐⭐⭐ |
| 10 | **配角抢戏** | 配角的篇幅超过主角，喧宾夺主 | ⭐⭐⭐ |

### 4.2 写作质量层面

| # | 弃书原因 | 具体表现 |
|---|----------|----------|
| 11 | **文笔生硬** | 对话不自然、叙述像流水账 |
| 12 | **错别字/病句** | 大量文字错误影响阅读体验 |
| 13 | **重复啰嗦** | 同一信息反复出现、同样的心理描写重复多次 |
| 14 | **视角混乱** | 随意切换POV、全知视角与限制视角混用 |
| 15 | **战斗/场景描写无力** | 关键场景写得干巴巴、缺乏画面感和紧张感 |

## 五、爽点（Satisfaction Point）理论

### 5.1 定义

"爽点"是指读者在阅读过程中获得的情绪满足——那种"看了真过瘾/真痛快"的感觉。

**爽点 = 压抑 + 释放**

没有压抑就没有爽感。压抑越深、释放越猛，爽感越强。

### 5.2 爽点的七大类型

| # | 爽点类型 | 描述 | 情绪机制 |
|---|----------|------|----------|
| 1 | **打脸型** | 被看不起 → 用实力打脸 | 压抑→翻身→解气 |
| 2 | **升级型** | 修炼/突破/获得新能力 | 期待→突破→满足 |
| 3 | **获宝型** | 获得珍贵宝物/传承/机缘 | 好奇→惊喜→兴奋 |
| 4 | **智谋型** | 精妙布局、反杀算计 | 困惑→恍然→佩服 |
| 5 | **逆袭型** | 绝境翻盘、弱胜强 | 绝望→燃→狂喜 |
| 6 | **揭秘型** | 真相大白、身份揭露 | 好奇→震撼→满足 |
| 7 | **情感型** | 深情告白、守护承诺 | 感动→心跳→幸福 |

### 5.3 爽点的节奏规则

```
【爽点密度公式】

章节类型        爽点密度要求
──────────────────────────────
开篇三章        每章至少1个明确爽点
正常推进        每2-3章至少1个小爽点
升级/突破章     本章必须有1个大爽点
打脸/对决章     本章必须有1个大爽点
过渡章          至少1个微爽点（小进展/小发现/幽默）
```

### 5.4 爽点设计的"三步法"

```
Step 1: 铺垫压抑（Build Tension）
  - 让主角遭遇不公/困境/挑战
  - 让读者产生"这太过分了"/"这怎么办"的情绪
  - 压抑时间：1-3章为宜，不超过5章

Step 2: 积蓄期待（Build Anticipation）
  - 暗示主角有能力/计划/准备
  - 让读者产生"等着看好戏"的期待
  - 期待时间：0.5-2章

Step 3: 爆发释放（Release/Payoff）
  - 主角采取行动，一举翻盘/打脸/突破
  - 结果要超出读者预期（比想象中更猛）
  - 释放时间：0.5-1章（要干脆利落，不拖泥带水）
```

### 5.5 每章爽点植入检查清单

对于每一章，至少满足以下一项：
- [ ] 主角展示了能力/智慧（让读者觉得主角厉害）
- [ ] 主角获得了某种进展（变强/获宝/推进目标）
- [ ] 主角赢得了一场冲突（打脸/说服/胜利）
- [ ] 揭露了一个有趣的信息（让读者"原来如此"）
- [ ] 角色关系产生了正向发展（结盟/认可/心动）
- [ ] 出现了让读者兴奋的新元素（新能力/新人物/新线索）

## 六、冲突-悬念-转折（CST）框架

### 6.1 框架定义

**CST框架**是网文章节设计的基础模型：

```
冲突（Conflict）→ 悬念（Suspense）→ 转折（Twist）
     ↓                  ↓                 ↓
  推动剧情          维持关注           制造惊喜
```

三者循环运转，构成网文的基本叙事引擎。

### 6.2 冲突（Conflict）的层次体系

```
Level 1：人与人冲突（最常用、最直接）
  - 与反派对抗
  - 与竞争者争夺
  - 与权威对抗
  - 误会/背叛导致的冲突

Level 2：人与环境冲突
  - 险境求生
  - 限时任务
  - 资源争夺
  - 规则限制

Level 3：人与自我冲突
  - 内心矛盾
  - 道德困境
  - 创伤克服
  - 身份认同

Level 4：势力与势力冲突
  - 门派之争
  - 国战
  - 阵营对抗
```

**冲突设计规则**：
1. 每章至少一个明确冲突源
2. 冲突要有"对等感"——对手不能太弱（没看头）也不能一直太强（太压抑）
3. 小冲突（1-3章解决）和大冲突（10-30章解决）交替推进
4. 冲突的解决方式要展现主角特点（武力型、智谋型、人格魅力型等）

### 6.3 悬念（Suspense）的设计方法

```
悬念的三层结构：

┌─ 微观悬念（章节级）
│  · "他打开门，看到的竟然是……"
│  · "这个人的真实身份是什么？"
│  · 作用：驱动读者翻下一章
│  · 周期：1-3章内揭晓
│
├─ 中观悬念（弧线级）
│  · "这次试炼的最终结果？"
│  · "主角能否突破到下一个境界？"
│  · 作用：维持一个叙事弧的张力
│  · 周期：10-30章内揭晓
│
└─ 宏观悬念（全书级）
   · "主角的身世之谜"
   · "最终BOSS的真实目的"
   · 作用：维持整本书的追读动力
   · 周期：全书或分阶段揭晓
```

**悬念管理原则**：
1. **三线并行**：同时保持至少1个微观 + 1个中观 + 1个宏观悬念
2. **揭而不尽**：解开一个悬念时，暗示更大的悬念
3. **及时兑现**：微观悬念不要超过5章不解开（读者会忘记）
4. **递进升级**：新悬念的吸引力 ≥ 已解开的悬念

### 6.4 转折（Twist）的类型与用法

| 转折类型 | 说明 | 适用场景 | 使用频率 |
|----------|------|----------|----------|
| **反转** | 事实与预期完全相反 | 真相揭露、阴谋反杀 | 每15-20章1次大反转 |
| **逆转** | 形势从劣转优或从优转劣 | 战斗、竞赛、谈判 | 每5-10章1次 |
| **意外** | 完全出乎意料的新要素 | 新角色登场、新能力觉醒 | 每10章1次 |
| **拐点** | 故事走向发生重大转变 | 阶段切换、目标改变 | 每30章1次 |
| **微转** | 小规模的出人意料 | 对话中的机锋、小发现 | 每1-2章1次 |

**转折设计规则**：
1. 大转折前必须有足够的伏笔（让读者回头看觉得合理）
2. 转折后要给"消化时间"——让角色和读者一起反应
3. 不要连续大转折（读者会疲劳/觉得假）
4. 转折要服务于主角成长和主线推进，不能为转而转

## 七、信息差（Information Asymmetry）技法

### 7.1 定义

"信息差"是指**不同角色之间、或角色与读者之间**存在的信息不对等。它是网文制造紧张感、优越感和期待感的核心技法。

### 7.2 三种信息差模式

```
模式A：读者 > 角色（上帝视角优越感）
┌──────────────────────────────────┐
│ 读者知道真相，角色不知道          │
│ 效果：读者产生优越感+紧张感       │
│ 典型："读者知道身边的人是叛徒"     │
│ 情绪：焦急、揪心、期待揭露时刻   │
└──────────────────────────────────┘

模式B：主角 > 其他角色（装逼打脸式）
┌──────────────────────────────────┐
│ 主角知道但配角/反派不知道         │
│ 效果：读者期待"你们等着被打脸"   │
│ 典型："反派不知道主角已经突破了"  │
│ 情绪：期待、痛快、满足           │
└──────────────────────────────────┘

模式C：角色 > 读者（悬念驱动式）
┌──────────────────────────────────┐
│ 角色知道某些事但读者不知道       │
│ 效果：制造悬念和好奇心          │
│ 典型："他露出了意味深长的微笑"   │
│ 情绪：好奇、紧张、期待揭秘     │
└──────────────────────────────────┘
```

### 7.3 信息差的实战技法

**技法1：延迟揭露（Delayed Reveal）**
- 方法：在第N章埋下线索，在第N+X章揭露真相
- 效果：读者恍然大悟，产生强烈满足感
- 关键：线索要足够但不明显，揭露时读者能回忆起来

**技法2：认知翻转（Cognitive Reversal）**
- 方法：先让读者/角色建立一个错误认知，再用真相打脸
- 效果：震撼 + 佩服作者的布局
- 关键：错误认知要建立得足够牢固，翻转才有力

**技法3：层叠信息差（Layered Asymmetry）**
- 方法：读者以为知道真相了，但真相之下还有真相
- 效果：二次/三次反转，持续制造惊喜
- 关键：每层真相都要合理

**技法4：观众席效应（Audience Effect）**
- 方法：有其他角色在场"围观"，但不知道主角的真实实力
- 效果：放大打脸/装逼的爽感
- 关键：围观者的反应要写得生动（震惊、不敢相信、跪服）

### 7.4 信息差管理清单

对于每一章，检查：
- [ ] 当前有哪些角色间的信息差在运作？
- [ ] 有没有新的信息差被建立？
- [ ] 有没有旧的信息差被消解？（适时揭露，不要积压太多）
- [ ] 读者此刻比主角多知道什么？（紧张感来源）
- [ ] 读者此刻比主角少知道什么？（悬念来源）
- [ ] 主角比对手多知道什么？（优越感/期待来源）

## 八、不同章节类型的节奏原则

### 8.1 章节类型分类

```
开篇章（第1-3章）     节奏：极快
上升章（进入新弧线）   节奏：快→中
推进章（发展中段）     节奏：中速稳健
过渡章（弧线间衔接）   节奏：中→慢→快
高潮章（弧线顶点）     节奏：极快爆发
收束章（弧线结尾）     节奏：快→渐缓
```

### 8.2 各类章节的写法规则

#### 开篇章（Opening Chapters）
```
节奏：⚡极快
字数分配：
  - 背景交代 ≤ 15%（融入行动中，不单独说明）
  - 冲突/行动 ≥ 50%
  - 角色刻画 ≈ 20%（通过行动和选择展示）
  - 悬念/钩子 ≈ 15%

关键原则：
  ✅ 开头就是事件，不是介绍
  ✅ 主角第一句话/第一个动作要有个性
  ✅ 500字内必须出现异常/冲突/变化
  ✅ 设定融入行动，别停下来解释
  ❌ 不要"某年某月某日"式开头
  ❌ 不要"我叫XXX，今年XX岁"式自我介绍
  ❌ 不要用描写天气/风景开头
  ❌ 不要先介绍世界观再讲故事
```

#### 上升章（Rising Action Chapters）
```
节奏：🔥快→中
字数分配：
  - 挑战/冲突升级 ≈ 40%
  - 主角应对/成长 ≈ 30%
  - 新信息/设定 ≈ 15%
  - 人物关系 ≈ 15%

关键原则：
  ✅ 每章至少推进一个冲突维度
  ✅ 展示对手的实力（让后面打赢更爽）
  ✅ 逐步释放主角的能力/筹码
  ✅ 保持紧迫感（倒计时/追兵/deadline）
  ❌ 不要一次性展示主角全部能力
  ❌ 不要让挑战太容易（没紧张感）
  ❌ 不要连续升级不给读者喘息
```

#### 高潮章（Climax Chapters）
```
节奏：⚡⚡极快爆发
字数分配：
  - 核心对决/事件 ≥ 60%
  - 情绪渲染 ≈ 20%
  - 围观反应 ≈ 10%
  - 转折/爆发 ≈ 10%

关键原则：
  ✅ 节奏紧凑，短句为主
  ✅ 多用动词，少用形容词
  ✅ 此前铺垫的伏笔在此触发
  ✅ 结果要超出预期（比读者想的更猛）
  ✅ 围观者的反应放大爽感
  ❌ 不要在高潮时插入长段回忆/内心独白
  ❌ 不要在关键时刻切到其他视角
  ❌ 不要虎头蛇尾（高潮结束太草率）
```

#### 过渡章（Transition Chapters）
```
节奏：⏸ 中→慢→快
字数分配：
  - 上一弧线的余波处理 ≈ 20%
  - 角色互动/轻松日常 ≈ 30%
  - 伏笔/新线索 ≈ 25%
  - 新弧线的钩子 ≈ 25%

关键原则：
  ✅ 最多2章过渡，第3章必须进入新冲突
  ✅ 过渡章也要有"微爽点"（小幽默/小甜蜜/小进展）
  ✅ 利用过渡章展示主角成长后的"新常态"
  ✅ 在过渡中自然埋入下一弧线的引子
  ❌ 不要纯日常超过1章（读者会无聊）
  ❌ 不要突然降速太多（读者适应不了）
  ❌ 不要把所有伏笔都集中在过渡章（太刻意）
```

### 8.3 节奏控制的"心电图"模型

理想的网文节奏应该像心电图，有规律地起伏：

```
情绪强度
  ↑
  │    ╱╲        ╱╲         ╱╲╱╲
  │   ╱  ╲    ╱╲╱  ╲     ╱╱    ╲╲
  │  ╱    ╲  ╱      ╲   ╱        ╲
  │╱╱      ╲╱        ╲╱╱          ╲╲
  │
  └──────────────────────────────────→ 章节
   1  3  5  7  10  15  20  25  30
   
   开篇 → 小高潮 → 过渡 → 中高潮 → 过渡 → 大高潮
```

**关键规则**：
- 低谷不能低于读者的"弃书线"（即使最平淡的章节也要有期待感）
- 高峰应该逐渐升高（后面的高潮要比前面更精彩）
- 低谷持续时间不超过2-3章
- 高潮持续时间3-5章为宜

## 九、可嵌入AI写作Prompt的规则摘要

以下是从上述方法论中提炼的、可直接嵌入AI写作系统prompt的核心规则：

### 9.1 章节生成通用规则

```
【网文章节生成核心规则】

1. 冲突优先：每章必须包含至少1个明确冲突（人vs人/人vs环境/人vs自我）
2. 章末钩子：每章最后200-300字必须留下悬念或引出下一章的期待
3. 爽点节奏：每2-3章至少1个明确爽点（打脸/升级/获宝/逆袭/揭秘）
4. 主角主导：主角在每章中必须是推动剧情的主体，不能沦为旁观者
5. 信息差运用：至少维持1个角色间信息差，制造期待感和紧张感
6. 进展感：每章至少推进1个维度（实力/关系/情报/目标/地点）
7. 对话占比：对话应占章节30%-50%，推动剧情而非空聊
8. 禁止注水：不允许无意义的重复描写、过长的心理独白或与主线无关的内容
```

### 9.2 开篇章节专用规则

```
【开篇章节（前3章）特殊规则】

1. 500字内必须出现冲突或异常事件
2. 禁止大段设定介绍，所有设定融入行动中
3. 第一章必须展示主角的性格特点和行动力
4. 三章内必须完成第一个"爽点循环"（压抑→积蓄→爆发）
5. 三章内出场角色不超过5个
6. 每章结尾必须有强钩子
7. 三章内必须让读者明确知道本书的核心卖点
```

### 9.3 爽点设计规则

```
【爽点设计规则】

1. 爽点公式：压抑（1-3章）→ 积蓄期待（0.5-2章）→ 爆发释放（0.5-1章）
2. 压抑不超过5章，否则读者弃书
3. 释放要干脆利落，不拖泥带水
4. 爆发程度要超出预期（比读者想象中更猛）
5. 有围观者的场景效果更好（围观者的震惊反应放大爽感）
6. 打脸前要先让对手足够嚣张（嚣张越过分，打脸越痛快）
7. 每个爽点都要推进主角成长或主线进展（不能空爽）
```

### 9.4 悬念与转折规则

```
【悬念与转折规则】

1. 三线悬念并行：始终保持微观（1-3章级）+ 中观（10-30章级）+ 宏观（全书级）悬念
2. 解开悬念时立即建立新悬念
3. 微观悬念不超过5章不揭晓
4. 大反转必须有前置伏笔（揭晓时读者觉得"原来如此"而不是"莫名其妙"）
5. 不连续使用同类型转折（避免审美疲劳）
6. 章末微转折/悬念是标配，大转折每15-20章一次
```

## 十、质量自检清单（QC Checklist）

每章完成后，用以下清单检查：

```
【强制项（必须全部满足）】
□ 本章有明确的冲突或紧张感？
□ 主角有主动行动（不是纯被动或旁观）？
□ 章末有钩子（读者有理由翻下一章）？
□ 不存在超过500字的纯设定/解释段落？
□ 没有逻辑漏洞或与前文矛盾？
□ 主角的行为符合已建立的人设？

【推荐项（至少满足3项）】
□ 有爽点或微爽点？
□ 运用了信息差技法？
□ 有新信息释放（推进读者理解）？
□ 角色关系有推进？
□ 有情绪变化（不是通篇一个情绪）？
□ 出现了转折或意外元素？

【禁止项（不能出现任何一项）】
□ 连续超过300字的设定介绍？
□ 主角做出与智商不符的愚蠢决定？
□ 无冲突的纯日常闲聊超过500字？
□ 与前文已确立的设定矛盾？
□ 重复前面已经写过的信息/描写？
□ 在高潮场景中插入冗长回忆？
```
